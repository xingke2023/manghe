# 约会盲盒 - 项目需求简要总结

## 一、项目定位

**产品名称：** 约会盲盒（"不正经实验室"旗下）

**核心定位：** 通过"盲盒+兴趣匹配+线下履约"机制，为都市青年提供高质量真实社交的小程序平台

**目标用户：** 20-35岁一二线城市单身男女、斜杠青年、自由职业者

---

## 二、核心亮点

### 1. 盲盒机制
- 降低社交焦虑和选择压力
- 增加约会惊喜感和新鲜感
- 通过"拆盒"营造游戏化体验

### 2. 价值观筛选
- 10题价值观测试
- 高风险选项自动触发人工审核
- 从源头保证社区质量

### 3. 双重履约保障
- **发盒者**：缴纳保证金
- **拆盒者**：支付防鸽费
- GPS打卡 + 双向二维码核销
- 智能判责与资金流转

### 4. 发盒凭证控制
- 管理员生成凭证分发给种子用户
- 种子用户可转赠给朋友
- 7天有效期，一次性消耗
- 控制社区增长节奏和质量

---

## 三、核心业务模式

### 约会流程（一对多）

```
发盒者发布盲盒
    ↓
多个拆盒者报名（支付防鸽费）
    ↓
发盒者选择多人锁定
    ↓
线下约会（GPS打卡 + 双向扫码）
    ↓
履约判定 + 资金结算
```

**关键特点：**
- ✅ 1个发盒者可以锁定多个拆盒者
- ✅ 每个拆盒者都需要与发盒者完成双向扫码
- ✅ 独立判定每个拆盒者的履约结果

---

## 四、核心功能模块

### 1. 用户注册与认证
- 微信一键登录
- AI头像生成（风格迁移技术）
- 基础信息 + 社交偏好设置
- 兴趣标签与照片上传

### 2. 盲盒发现
- **盲盒广场**：信息流 + 多维度筛选（时间/地区/分类）
- **推荐算法**：兴趣匹配度(60%) + 新近度(40%)
- **拆盒次数限制**：
  - 普通用户：3次/天
  - 会员用户：10次/天
  - 已拆过的盲盒可反复查看不扣次数
- **我关注的**：会员功能，关注感兴趣的发盒者

### 3. 盲盒发布
**三重门槛：**
1. 价值观测试通过
2. 缴纳保证金
3. 使用发盒凭证

**发布内容：**
- 封面、主题、时间、地点、费用类型
- 期待特质、体验价值
- 最多接受多少人（支持一对多）

### 4. 互动沟通
- **一句话盲聊**：拆盒者首次发言后需等待发盒者回复
- **聊天弹窗**：在详情页即可沟通
- **资料权限申请**：拆盒者需申请查看发盒者兴趣照
- **会话销毁**：活动结束2小时后自动销毁

### 5. 履约核销（核心创新）

#### 线下打卡
- GPS定位验证（300米范围内）
- 时间窗口：活动前3小时至活动后2小时

#### 双向见面码核销
- 每个参与者生成唯一二维码
- 发盒者需扫描每个拆盒者的码
- 每个拆盒者需扫描发盒者的码
- 双向扫描完成才算履约成功

#### 智能判责
| 发盒者 | 拆盒者 | 双向扫码 | 判定结果 | 资金处理 |
|--------|--------|----------|----------|----------|
| 已打卡 | 已打卡 | 已完成   | 完美履约 | 双方费用退还 |
| 已打卡 | 未打卡 | -        | 拆盒者失约 | 扣拆盒者防鸽费，50%给发盒者 |
| 未打卡 | 已打卡 | -        | 发盒者失约 | 扣发盒者保证金，50%给拆盒者 |
| 未打卡 | 未打卡 | -        | 双方失约 | 双方费用归平台 |
| 已打卡 | 已打卡 | 未完成   | 已打卡未核销 | 默认双方失约，可申诉 |

### 6. 个人中心
- 个人主页（资料编辑、会员状态）
- 进行中的盲盒
- 我的履约、发盒记录、关注的TA
- 系统设置

### 7. 管理后台
- 用户管理
- 价值观测试审核
- 发盒凭证生成与分发
- 盲盒内容监控
- 履约申诉处理

## 六、关键业务规则

### 1. 拆盒次数限制
- 普通用户：3次/天，会员：10次/天
- 已拆过的盲盒可反复查看不扣次数
- 只有首次拆某个盲盒时才扣除当日次数

**实现逻辑：**
```
1. 检查 box_views 是否拆过此盲盒
2. 已拆过 → 直接显示（不扣次数）
3. 未拆过 → 检查今日已用次数
4. 未超限 → 记录到 box_views + 今日统计+1
```

### 2. 匹配与锁定
- 一个盲盒可以锁定多个拆盒者
- 通过 `box_applications.is_locked = 1` 标记
- 未被选中的报名者状态变为"擦肩而过"

### 3. 履约判定
- 基于 `checkins` 表判断是否打卡
- 基于 `meeting_verifications` 表判断是否双向扫码
- 每个拆盒者独立判定履约结果
- 自动触发资金流转

### 4. 系统通知（11类触点）
- 报名成功、匹配成功/失败
- 盲盒下架/过期
- 收到新消息、资料查看权限通过
- 收到新报名、约会开始前提醒
- 履约结果通知、资金变动通知
- 发盒资格审核结果

## 七、技术要点

### 1. AI头像生成
- 使用风格迁移技术（非大型AIGC）
- 用户上传照片 + 官方风格图
- 生成2张风格化头像供选择

### 2. GPS定位
- 使用高德/百度地图SDK
- 打卡时验证与约定地点距离 ≤ 300米
- 记录打卡坐标作为履约凭证

### 3. 二维码核销
- 每个参与者生成唯一二维码
- 24小时有效期
- 防止重复扫码（唯一约束）

### 4. 推荐算法
```
推荐分 = 兴趣匹配度(60%) + 新近度(40%)

兴趣匹配度 = (共同标签数 / 用户总标签数) × 100
新近度：24小时内100分，24-48小时90分，48-72小时80分
```

### 5. 聊天记录销毁
- 活动结束后2小时自动销毁
- 定时任务执行删除
- 保护用户隐私

## 十一、核心数据流

### 发盒流程
```
通过价值观测试 → 缴纳保证金 → 输入凭证 → 填写内容 → 发布
```

### 拆盒流程
```
拆盒（检查次数）→ 查看详情 → 聊天 → 报名（支付防鸽费）→ 等待
```

### 匹配流程
```
发盒者锁定多人 → 更新 box_applications.is_locked = 1
→ 拒绝其他人 → 生成约会记录
```

### 履约流程
```
打卡（GPS）→ 生成见面码 → 双向扫码 → 履约判定 → 资金结算
```

## 十二、风险控制

### 1. 内容风险
- 价值观测试双重把关（自动+人工）
- 管理后台可强制下架违规内容

### 2. 履约风险
- 保证金+防鸽费双重约束
- GPS+双向扫码技术验证
- 智能判责+申诉机制

### 3. 隐私保护
- 聊天记录定时销毁
- 地点信息匹配前保密
- 资料查看需申请授权

### 4. 平台责任
- 申诉处理机制
- 人工可干预修正判定
- 用户冻结/解冻功能
