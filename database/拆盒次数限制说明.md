# 拆盒次数限制功能说明（简化版）

## 业务需求

**核心规则：**
1. 用户每天有**拆盒次数限制**：普通用户3次/天，会员10次/天
2. 用户**之前拆过的盲盒**可以反复查看，不扣除次数
3. 只有**首次拆某个盲盒**时才扣除当日次数

---

## 数据库设计

### 1. box_views 表 - 拆盒记录表

记录用户**所有拆过的盲盒**（不限日期）。

```sql
CREATE TABLE `box_views` (
  `id` BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  `box_id` BIGINT UNSIGNED NOT NULL,           -- 盲盒ID
  `viewer_id` BIGINT UNSIGNED NOT NULL,        -- 查看者ID
  `first_view_at` DATETIME DEFAULT CURRENT_TIMESTAMP,  -- 首次查看时间

  UNIQUE KEY `uk_box_viewer` (`box_id`, `viewer_id`)
);
```

**作用：**
- 记录用户拆过哪些盲盒
- 判断是否首次拆盒
- **永久保留**（不按日期清理）

---

### 2. daily_box_view_stats 表 - 每日拆盒统计表

记录用户每天拆了多少个**新盲盒**。

```sql
CREATE TABLE `daily_box_view_stats` (
  `id` BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT UNSIGNED NOT NULL,          -- 用户ID
  `stat_date` DATE NOT NULL,                   -- 统计日期
  `view_count` INT UNSIGNED DEFAULT 0,         -- 当日已拆新盒次数

  UNIQUE KEY `uk_user_date` (`user_id`, `stat_date`)
);
```

**作用：**
- 快速查询用户今日已用次数
- 每日一条记录，次日自动重置
- 只记录**首次拆新盒**的次数

---

## 业务逻辑

### 拆盒流程图

```
用户点击盲盒
    ↓
检查 box_views 是否有记录 (box_id + viewer_id)
    ↓
已存在？
├─ 是 → 直接显示详情（不扣次数）✅
└─ 否 → 继续检查
        ↓
    查询 daily_box_view_stats 今日已用次数
        ↓
    已用次数 >= 限制？
    ├─ 是 → 提示"今日次数已用完" ❌
    └─ 否 → 允许拆盒
            ↓
        1. INSERT INTO box_views (永久记录)
        2. UPDATE daily_box_view_stats view_count +1
        3. 显示盲盒详情 ✅
```

## SQL实现

### 核心查询逻辑

```sql
-- 步骤1: 检查是否拆过此盲盒
SELECT COUNT(*) as already_viewed
FROM box_views
WHERE box_id = ? AND viewer_id = ?;

IF already_viewed > 0 THEN
    -- 已拆过，直接显示
    RETURN SHOW_DETAIL;
END IF;

-- 步骤2: 首次拆盒，检查今日次数
SELECT IFNULL(view_count, 0) as used_count
FROM daily_box_view_stats
WHERE user_id = ? AND stat_date = CURDATE();

-- 步骤3: 判断是否超限
SELECT is_member FROM users WHERE id = ?;
limit = is_member ? 10 : 3;

IF used_count >= limit THEN
    RETURN ERROR('今日拆盒次数已用完');
END IF;

-- 步骤4: 允许拆盒，记录数据
-- 记录拆盒（永久）
INSERT INTO box_views (box_id, viewer_id) VALUES (?, ?);

-- 更新今日统计
INSERT INTO daily_box_view_stats (user_id, stat_date, view_count)
VALUES (?, CURDATE(), 1)
ON DUPLICATE KEY UPDATE view_count = view_count + 1;

-- 显示详情
RETURN SHOW_DETAIL;
```

## 应用层伪代码

```javascript
async function viewBoxDetail(userId, boxId) {
  // 1. 检查是否拆过
  const alreadyViewed = await BoxView.exists({
    box_id: boxId,
    viewer_id: userId
  });

  if (alreadyViewed) {
    // 已拆过，直接返回详情
    return await getBoxDetail(boxId);
  }

  // 2. 首次拆盒，检查今日次数
  const user = await User.findById(userId);
  const limit = user.is_member ? 10 : 3;

  let stat = await DailyBoxViewStat.findOne({
    user_id: userId,
    stat_date: today()
  });

  const usedCount = stat?.view_count || 0;

  if (usedCount >= limit) {
    throw new Error('今日拆盒次数已用完，明天再来吧！');
  }

  // 3. 允许拆盒，记录数据
  // 永久记录
  await BoxView.create({
    box_id: boxId,
    viewer_id: userId
  });

  // 更新今日统计
  if (stat) {
    stat.view_count += 1;
    await stat.save();
  } else {
    await DailyBoxViewStat.create({
      user_id: userId,
      stat_date: today(),
      view_count: 1
    });
  }

  // 更新盲盒浏览数
  await BlindBox.increment('view_count', { where: { id: boxId } });

  return await getBoxDetail(boxId);
}
```

## 优化建议

### 1. Redis缓存今日次数

```javascript
// 缓存Key: daily_view:{user_id}:{date}
const cacheKey = `daily_view:${userId}:${today()}`;

// 读取
let count = await redis.get(cacheKey);
if (count === null) {
  count = await DB.query(...); // 从数据库读取
  await redis.setex(cacheKey, 86400, count); // 缓存到当天结束
}

// 更新
await redis.incr(cacheKey);
```

### 2. 批量查询已拆盒子

```javascript
// 一次查询用户拆过的所有盒子
const viewedBoxIds = await BoxView.findAll({
  where: { viewer_id: userId },
  attributes: ['box_id']
});

// 返回给前端标记
return { boxes, viewed_box_ids: viewedBoxIds };
```

## 定时任务

### 每日零点：清理旧数据

```sql
-- 清理30天前的统计数据
DELETE FROM daily_box_view_stats
WHERE stat_date < DATE_SUB(CURDATE(), INTERVAL 30 DAY);
```

**注意：** `box_views` 表**永久保留**，不清理。

## 数据分析

### 1. 今日活跃用户

```sql
SELECT COUNT(DISTINCT user_id) as active_users
FROM daily_box_view_stats
WHERE stat_date = CURDATE();
```

### 2. 次数利用率

```sql
-- 统计今日用户拆盒次数分布
SELECT
  view_count,
  COUNT(*) as user_count,
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(), 2) as percentage
FROM daily_box_view_stats
WHERE stat_date = CURDATE()
GROUP BY view_count
ORDER BY view_count;

-- 结果示例：
-- view_count | user_count | percentage
-- -----------|------------|------------
--     1      |    200     |   45.45%
--     2      |    150     |   34.09%
--     3      |     90     |   20.45%   ← 用完3次的用户
```

### 3. 会员转化机会

```sql
-- 达到上限的普通用户（潜在会员）
SELECT COUNT(*) as potential_vip_users
FROM daily_box_view_stats s
JOIN users u ON s.user_id = u.id
WHERE s.stat_date = CURDATE()
  AND s.view_count >= 3
  AND u.is_member = 0;
```

## 总结

**设计优势：**
✅ **简单高效**：只需两张表，逻辑清晰
✅ **永久记录**：用户拆过的盒子永久可看
✅ **精准限制**：只有首次拆新盒才扣次数
✅ **自动重置**：次日自动获得新额度
✅ **会员权益**：支持差异化次数限制

**核心要点：**
- `box_views`：记录**所有**拆过的盲盒（永久）
- `daily_box_view_stats`：记录**今日**拆了多少**新盒**（临时）
- 重复查看不扣次数 = 提升用户体验
- 每日限制新盒数量 = 控制平台负载
